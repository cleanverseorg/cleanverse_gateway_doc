Cleanverse Gateway API Documentation

This documentation provides details about the Cleanverse Gateway API endpoints, including request formats, parameters, and response examples.

POST /api/gateway/user_register

This endpoint is used to register a new user. It requires user identity information, account details, and wallet information.

Request Headers

Parameter Type Required Description
Accept String Yes Must be set to "application/json"
api-key String Yes API key obtained from environment variables
api-id String Yes API ID obtained from environment variables

Request Body

Parameter Type Required Description
fullName String Yes User's full name
idType Integer Yes Identification type (1: ID Card;2: PASSPORT)
idNumber String Yes Identification number
validUntil String Yes ID expiration date in format: YYYY-MM-DD
countryISO2 String Yes Country code (ISO2 format)
accountNo String Yes Account number
walletAddress String Yes Wallet address
walletNetwork String Yes Wallet network (e.g., sol)

Request Example

{
   "fullName": "test",
   "idType": 1,
   "idNumber": "1234567890",
   "validUntil": "2033-10-18",
   "countryISO2": "sg",
   "accountNo": "12345567890",
   "walletAddress": "3aNLPZC2Z5PgLRBgnwQuSgnBXgeDjM6pAhErQeS87x99",
   "walletNetwork": "sol"
}

Response

200 Success

Successful response will return the registration result.

POST /api/gateway/get_deposit_address

This endpoint is used to retrieve deposit addresses for specified wallets and networks.

Request Headers

Parameter Type Required Description
Accept String Yes Must be set to "application/json"
api-key String Yes API key obtained from environment variables
api-id String Yes API ID obtained from environment variables

Request Body

Parameter Type Required Description
wallets Array Yes Array of wallet objects
wallets[].walletAddress String Yes Wallet address
wallets[].network String Yes Blockchain network (e.g., solana)

Request Example

{
    "wallets": [
        {
            "walletAddress": "8eEjtKzJbpDoeww1sXHADh4gHWjmfj6iYVbDm2k5y3Fx",
            "network": "solana"
        }
    ]
}

Response

200 Success

Successful response will return deposit address information for the specified wallets.

POST /api/gateway/build_withdraw

This endpoint is used to build a withdrawal transaction. It prepares the withdrawal request for submission.

Request Headers

Parameter Type Required Description
Accept String Yes Must be set to "application/json"
api-key String Yes API key obtained from environment variables
api-id String Yes API ID obtained from environment variables

Request Body

Parameter Type Required Description
userPublicKey String Yes User public key address
amount String Yes Withdrawal amount

Request Example

{
  "userPublicKey": "6wT4QwM3it2m4vso7W8M1t2c23Pq3msqYxVqE2JjQhHh",
  "amount": 1000000
}

Response

200 Success

Successful response will return deposit address information for the specified wallets.

POST /api/gateway/submit_withdraw

This endpoint is used to submit a previously built withdrawal transaction for processing.

Request Headers

Parameter Type Required Description
Accept String Yes Must be set to "application/json"
api-key String Yes API key obtained from environment variables
api-id String Yes API ID obtained from environment variables

Request Body

Parameter Type Required Description
userPublicKey String Yes User public key address
messageBase64 String Yes Messages encoded in Base64
userSignatureBase64 String Yes The user's Base64-encoded signature (64 bytes)

Request Example

{
  "userPublicKey": "6wT4QwM3it2m4vso7W8M1t2c23Pq3msqYxVqE2JjQhHh",
  "messageBase64": "AQIDBAUGBwgJ...",
  "userSignatureBase64": "BASE64_64_BYTE_SIGNATURE"
}

Response

200 Success

Successful response will return deposit address information for the specified wallets.

Authentication

All API requests require authentication using "api-key" and "api-id" provided in the request headers.

Note: The "api-key" and "api-id" should be kept secure and not exposed in client-side code.

Required Headers

Header Description Required Example
Accept application/json Yes application/json
api-key Your unique API key for authentication Yes 409ZvwsL4vSYwWdVTYOTF6TzoBfxCe1h
api-id Your API identifier Yes APP20251015152622BQH4L3

The API base URL varies depending on the environment you're working with.

Environments

Environment Base URL Purpose
